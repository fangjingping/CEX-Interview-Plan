# 工程实操演练（CTO 面试向）

## 规则
- 时间建议：2~4 小时。
- 输出要求：代码 + 测试 + 简要说明。

## 演练题目
1) 幂等写入与 TTL 去重  
在 `demo-middleware` 中增加幂等去重的 TTL 逻辑，并补充测试。

2) 订单簿快照与回放  
在 `demo-system-design` 中加入快照恢复后的一致性校验。

3) 强平价计算  
在 `demo-risk` 中实现强平价计算（含风险限额阶梯）。

4) 限流与熔断组合  
在 `demo-resilience` 中加入限流 + 熔断联合策略。

5) Kafka 分区顺序验证  
在 `demo-middleware` 中增加分区顺序可视化校验逻辑。

6) 资金费率结算模拟  
在 `demo-risk` 中增加资金费率对权益与保证金率的影响。

## 评分维度
- 正确性：边界条件、异常路径与幂等性。
- 性能意识：避免不必要分配与阻塞。
- 可维护性：命名、结构与测试覆盖。
