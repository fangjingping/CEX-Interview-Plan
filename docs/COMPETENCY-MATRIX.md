# 能力矩阵（CTO 面试向）

## 使用方式
用于统一评估标准，聚焦“能落地、可验证、可排障”的工程能力与业务理解深度。

## 评分维度与证据

| 维度 | 基础（必须） | 熟练（加分） | 专家（核心） | 证据与验证 |
| --- | --- | --- | --- | --- |
| Java 核心 | 集合/泛型/异常、I/O/NIO 基本使用 | 理解序列化/反射成本 | 能解释设计取舍与失败模式 | 代码片段、性能指标 |
| 并发与 JMM | 线程池与并发容器 | CAS/锁优化/背压 | 低延迟设计与瓶颈分析 | 线程栈、JFR/GC |
| JVM 与性能 | GC 基础与参数 | 定位 OOM/抖动 | safepoint/JIT 退化定位与优化 | GC log/火焰图 |
| 一致性与幂等 | 请求幂等与去重 | 事件溯源/回放 | 一致性边界与恢复方案 | 方案/演练记录 |
| 中间件治理 | Kafka/Redis/DB 基础 | 分区顺序/TTL/隔离级别 | 失败模式与回退 | 指标/压测 |
| 合约业务 | 保证金/强平/资金费率 | 仓位模式/风险限额 | 极端行情处理 | 场景推演 |
| 高可用运维 | 基本监控与告警 | 灰度/回滚/限流 | 演练与复盘闭环 | 演练与文档 |

## 淘汰红线
- 无法解释幂等与一致性边界。
- 无法说明强平/资金费率的核心逻辑。
- 无法用证据链定位延迟抖动或 OOM。
