# 面试评分标准（CTO 面试向）

## 评分档位
- 不合格：无法给出可验证结论或无明确边界。
- 合格：能解释关键概念与基本方案。
- 优秀：能说明权衡与失败模式。
- 专家：能提供恢复路径与演练策略。

## 维度评分

| 维度 | 合格 | 优秀 | 专家 |
| --- | --- | --- | --- |
| Java 并发/JVM | 能解释线程池与 GC | 能定位延迟与 OOM | 有完整排障闭环 |
| 一致性与幂等 | 能说明去重策略 | 能定义一致性边界 | 有回放与修复方案 |
| 中间件治理 | 能说明顺序与缓存 | 能描述失败模式 | 有监控与回退 |
| 合约业务 | 能解释保证金/强平 | 能说明资金费率与仓位模式 | 能推演极端行情 |
| 高可用运维 | 能描述限流与降级 | 能制定演练方案 | 能落地 SLO 与复盘 |

## 淘汰红线
- 无法解释合约强平与资金费率的核心逻辑。
- 无法说明幂等与一致性如何落地。
- 无法提供排障证据链。
