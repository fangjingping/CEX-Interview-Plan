# 清结算与对账手册（CTO 面试向）

## 目标
展示撮合、账本、钱包与报表之间的对账链路，保证资金守恒与可追溯。

## 核心能力清单
- 双录账本：分录粒度、费用、返佣与折扣策略。
- 结算流程：成交清算、资金费率结算、手续费归集。
- 对账体系：撮合 vs 账本 vs 钱包对账，差异处理与补偿。
- 报表与审计：日终/日内对账、异常追踪与修正机制。
- 费率与激励：Maker/Taker、返佣与活动补贴核算。
- 精度处理：费率与 PnL 的舍入与对账差异修正。

## 典型面试题与追问
必问
- 如何保证撮合结果与账本一致？差异如何处理？
- 资金费率结算如何对账与回滚？
追问
- 出现错账时的修复流程是什么？
- 审计与报表如何与业务流水对齐？

## 回答结构建议
先定义资金守恒与责任边界 → 描述清结算链路 → 对账与补偿 → 审计闭环。

## 常见陷阱
- 只描述撮合与账本，不谈对账与修复机制。
- 结算过程缺少审计证据链。
