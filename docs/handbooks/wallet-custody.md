# 钱包与托管手册（CTO 面试向）

## 目标
覆盖充值/提现与资产托管的关键流程，强调安全、风控与可追溯性。

## 核心能力清单
- 钱包体系：热钱包/冷钱包、多签与门限签名、资金归集与分层托管。
- 充值确认：确认数策略、链重组处理、孤块回滚。
- 提现流程：风控规则、人工审核、限额与灰度放行。
- 密钥管理：HSM/密钥轮换、权限隔离与操作审计。
- 地址治理：白名单、提现地址风控与标签化管理。
- 链上模型：UTXO/账户模型差异与链上监控。

## 典型面试题与追问
必问
- 提现全链路如何设计？如何避免资金被盗或误提？
- 充值确认与链重组如何处理？
追问
- 热/冷钱包迁移与异常资产如何处置？
- 大额提现如何做分层审核与风控？

## 回答结构建议
先定义安全边界与资产分层 → 描述充值/提现状态机 → 风控与审计 → 容灾与应急流程。

## 常见陷阱
- 只谈业务流程，不谈密钥与权限隔离。
- 忽略链上异常（回滚、拥堵、费用激增）。
