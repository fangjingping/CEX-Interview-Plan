# 账户体系与权限手册（CTO 面试向）

## 目标
覆盖用户、子账户、API Key 与权限治理，强调幂等、审计与资金隔离。

## 核心能力清单
- 账户模型：主/子账户、团队账号、机构账户层级。
- 权限体系：RBAC/ABAC、API Key 权限与范围控制。
- 资金操作：划拨、冻结/解冻、限额与风控策略。
- 审计与追踪：操作日志、审批流程与责任归属。
- 安全增强：MFA、IP 白名单、API Key 轮换与会话治理。

## 典型面试题与追问
必问
- API Key 权限如何设计？如何防止越权？
- 子账户资金隔离与授权如何落地？
追问
- 高风险操作如何做二次确认与审批？
- 如何处理权限变更后的历史行为审计？

## 回答结构建议
先定义账户边界与权限模型 → 描述关键操作路径 → 审计与回溯 → 风险控制。

## 常见陷阱
- 只谈认证，不谈授权与最小权限。
- 权限变更缺少审计与可追溯性。
