# 合规与运营治理手册（CTO 面试向）

## 目标
覆盖 KYC/AML、监管合规与运营治理，确保安全、可审计与可持续运营。

## 核心能力清单
- 合规体系：KYC/AML、制裁名单、Travel Rule、可疑交易上报。
- 法币出入金：PSP 对接、充值提现对账、拒付与风控策略。
- 安全治理：密钥管理、权限隔离、最小权限与审计留存。
- 运营治理：告警体系、事故流程、发布与回滚、容量规划。
- 市场监察：操纵识别、异常交易检测与证据链固化。
- 数据治理：WORM 审计、数据留存与隐私合规。

## 典型面试题与追问
必问
- KYC/AML 如何落地？合规规则变更如何灰度？
- 法币出入金的对账与风控如何设计？
追问
- 监管审计需要哪些证据链？
- 事故复盘与持续改进如何制度化？

## 回答结构建议
先说明合规边界与监管要求 → 描述系统流程 → 审计与证据链 → 运营治理。

## 常见陷阱
- 将合规视为纯流程，缺少系统化落地与证据链。
- 忽视法币链路的对账与拒付风险。

## 演示关联
- 暂无独立演示模块，可结合 `gateway-auth` 与 `wallet-custody` 讨论合规落地。
