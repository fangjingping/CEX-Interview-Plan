# Spring Cloud 治理手册（CTO 面试向）

## 目标
覆盖注册中心、配置中心、网关与弹性治理（Spring Boot 3 / Spring Cloud 2023+），强调稳定性、可观测性与故障隔离。

## 考核原则（CTO 视角）
- 能否在高并发与高可用下保持服务稳定。
- 能否基于指标与故障场景做出治理决策。
- 能否明确限流、降级、熔断与重试的边界。

## 应用场景（CEX）
- API 网关：鉴权、限流、灰度发布、熔断降级。
- 撮合/风控/账本服务的注册发现与路由。
- 配置中心：风控参数、限额、费率与灰度策略。

## 核心能力清单
- 注册中心：服务发现、健康检查、元数据路由（Nacos/Eureka/Consul）。
- 配置中心：动态配置、版本回滚、灰度发布（Spring Cloud Config/Nacos）。
- 网关治理：鉴权、限流、熔断、降级与重试（Spring Cloud Gateway/Resilience4j）。
- 可观测：Tracing/Metrics/Logging 的链路闭环（Micrometer Tracing/OTel/Prometheus）。

## 典型面试题与追问
必问
- 限流与熔断的策略如何设计？如何避免误伤？
- 配置中心的灰度与回滚如何落地？
追问
- 注册中心抖动如何避免大规模雪崩？
- 多区域部署时如何做就近路由？

## 回答结构建议
先定义稳定性目标 → 描述治理策略 → 失败模式与降级 → 可观测与回滚。

## 常见陷阱
- 只讲组件名，不讲治理策略与阈值依据。
- 重试与熔断策略互相放大故障。
